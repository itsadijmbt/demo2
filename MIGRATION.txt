  2 Lifespan body inlined into main()
     fastmcp's `lifespan=create_lifespan(args)` ran the lifespan
     coroutine automatically at startup -- it built SnowflakeService,
     registered tools/resources, attached middleware, then yielded.

     SecureMCP has no `lifespan=` kwarg. We do that work by hand in
     main() in this exact order:

         connection_params = {...}
         snowflake_service = SnowflakeService(...)   # was inside lifespan
         server = SecureMCP("snowflake-mcp")
         initialize_tools(snowflake_service, server)
         initialize_resources(snowflake_service, server)
         install_query_check(server, allow, deny)    # see (d)
         try:
             server.run()
         finally:
             cleanup_snowflake_service(snowflake_service)

     The `create_lifespan` function is left in the file as dead code
     for diff visibility (per the project's comment-don't-delete rule).
